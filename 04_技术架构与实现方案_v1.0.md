### 1.2 请求流转路径

```
【用户浏览器】
      ↓ 输入 https://domain.com
【Nginx 80/443】
      ├── 匹配 / → 返回 React 静态资源（index.html + assets）
      └── 匹配 /api → proxy_pass 到 FastAPI（localhost:8000）
            ↓
【FastAPI 服务】
      ├── 中间件链：CORS → 限流 → JWT 鉴权 → 请求日志
      └── 路由分发 → 业务处理 → SQLAlchemy ORM
            ↓
【数据层】SQLite 读写 / 本地文件写入
```

| 环境 | 数据库 | 说明 |

### 3.2 模块职责说明

#### 3.2.1 API 网关层

| 中间件 | 职责 | 实现方式 |
|--------|------|----------|
| CORS | 跨域请求处理 | FastAPI CORSMiddleware，允许管理后台域名 |
| RateLimit | 接口限流 | 基于内存计数器，按 IP + 接口维度限流 |
| JWT Auth | 统一鉴权 | Authorization: Bearer {token}，校验管理员身份 |
| RequestLog | 请求日志 | 记录请求路径、方法、耗时、状态码、操作人 |
| ExceptionHandler | 异常统一处理 | 业务异常 → 400/403；系统异常 → 500；统一返回格式 |

#### 3.2.2 回收服务（Recycle Service）

| 功能 | 说明 |
|------|------|
| 旧件上传 | 接收师傅或管理员上传的旧件照片、分类、描述信息 |
| 回收列表 | 支持多维度筛选（状态、分类、时间、师傅、关键词）和分页 |
| 回收详情 | 展示回收单完整信息、照片、审核记录、操作日志 |
| 审核操作 | 管理员对回收单进行审核：通过 / 驳回 / 待补充 |
| 入库确认 | 仓库管理员确认旧件实物已入库，状态流转为「已入库」 |
| 积分发放 | 触发奖励服务，按规则计算并发放积分/奖金 |
| 导出报表 | 按筛选条件导出 Excel 回收报表 |

#### 3.2.3 师傅服务（Master Service）

| 功能 | 说明 |
|------|------|
| 列表查询 | 师傅信息列表，支持姓名、手机号、状态、资质等级筛选 |
| 详情查看 | 师傅个人信息、资质、历史回收记录、累计奖励 |
| 启用禁用 | 控制师傅账号状态，禁用后无法上传旧件 |
| 资质审核 | 审核师傅提交的资质证明材料 |
| 数据统计 | 师傅维度的回收数量、积分、排名统计 |

#### 3.2.4 奖励服务（Award Service）

| 功能 | 说明 |
|------|------|
| 积分发放 | 按回收单审核结果，根据配置规则计算积分并发放 |
| 发放记录 | 查询某师傅或全局的积分发放历史 |
| 统计汇总 | 按时间段统计发放总额、发放笔数、平均积分 |
| 批量发放 | 批量审核通过并发放积分（后台批量操作） |

#### 3.2.5 配置服务（Config Service）

| 功能 | 说明 |
|------|------|
| 参数配置 | 系统级参数：积分规则、回收品类、审核阈值等 |
| 分类管理 | 废旧品类增删改查（家电、数码、金属、塑料等） |
| 回收规则 | 各品类对应的积分兑换规则、重量/数量换算 |
| 通知模板 | 审核结果通知、积分到账通知的模板配置 |

#### 3.2.6 看板服务（Dashboard Service）

| 功能 | 说明 |
|------|------|
| 汇总统计 | 今日/本周/本月回收数量、待审核数、已入库数、积分发放数 |
| 趋势分析 | 按日/周/月的回收量、入库量、审核量趋势折线图数据 |
| 分类占比 | 各品类回收数量/重量占比饼图数据 |
| 师傅排行 | Top N 师傅的回收贡献排行榜 |

#### 3.2.7 文件服务（Upload Service）

| 功能 | 说明 |
|------|------|
| 图片上传 | 接收 multipart/form-data 图片，校验格式和大小 |
| 图片读取 | 通过 URL 访问已上传的图片（缩略图 + 原图） |
| 文件下载 | 导出报表文件的下载接口 |
| 缩略图 | 自动生成压缩缩略图，减少前端加载耗时 |

## 4. 接口设计

### 4.1 通用规范

#### 4.1.1 通信协议

| 项目 | 规范 |
|------|------|
| 协议 | HTTPS |
| 数据格式 | JSON |
| 编码 | UTF-8 |
| 请求方式 | RESTful：GET / POST / PUT / DELETE |
| 鉴权方式 | Header: `Authorization: Bearer {jwt_token}` |

#### 4.1.2 统一返回格式

```json
{
  "code": 200,
  "msg": "操作成功",
  "data": {}
}
```

| 字段 | 类型 | 说明 |
|------|------|------|
| code | int | 业务状态码：200 成功，400 参数错误，401 未登录，403 无权限，500 服务异常 |
| msg | string | 返回消息，成功为"操作成功"，失败为具体错误描述 |
| data | object / array | 实际业务数据，可为空对象或空数组 |

#### 4.1.3 全局错误码

| 错误码 | 含义 | 触发场景 |
|--------|------|----------|
| 200 | 成功 | 请求正常处理完毕 |
| 400 | 参数错误 | 必填字段缺失、格式非法、参数范围超限 |
| 401 | 未登录 | Token 缺失、Token 过期、Token 无效 |
| 403 | 无权限 | 无操作权限、账号被禁用、非管理员 |
| 404 | 资源不存在 | 回收单/师傅/配置项 ID 不存在 |
| 409 | 业务冲突 | 状态不允许操作（如已入库的回收单再次审核） |
| 429 | 请求频繁 | 触发限流规则 |
| 500 | 服务异常 | 服务端内部错误 |

#### 4.1.4 分页规范

```json
{
  "code": 200,
  "msg": "查询成功",
  "data": {
    "list": [...],
    "pagination": {
      "page": 1,
      "page_size": 20,
      "total": 156,
      "total_pages": 8
    }
  }
}
```

| 参数 | 类型 | 说明 |
|------|------|------|
| page | int | 当前页码，默认 1 |
| page_size | int | 每页条数，默认 20，最大 100 |
| total | int | 总记录数 |
| total_pages | int | 总页数 |

## 5. 安全规范

### 5.1 鉴权机制

#### 5.1.1 JWT Token 鉴权

| 项目 | 规范 |
|------|------|
| Token 生成 | 登录成功后服务端签发 JWT Token |
| Token 传输 | Header: `Authorization: Bearer {token}` |
| Token 有效期 | Access Token: 2 小时；Refresh Token: 7 天 |
| Token 存储 | 前端存储于 localStorage，请求时自动注入 Header |
| Token 刷新 | 接口返回 401 时，自动调用 refresh 接口换取新 Token |
| Token 算法 | HS256，密钥由服务端随机生成，存储于环境变量 |
| Token 载荷 | `{ sub: admin_id, username, role, iat, exp }` |

#### 5.1.2 权限控制

| 角色 | 权限范围 |
|------|----------|
| super_admin | 全部权限，包括删除操作、配置修改、管理员管理 |
| admin | 回收单审核、入库确认、积分发放、师傅管理、数据查看 |
| auditor | 仅回收单审核、查看 |
| storekeeper | 仅入库确认、回收单查看 |

权限校验在 JWT 中间件之后，通过 `@require_role` 装饰器控制接口访问。

### 5.2 接口限流

| 维度 | 规则 | 实现 |
|------|------|------|
| 全局限流 | 单 IP 60 秒内最多 100 次请求 | 内存计数器，滑动窗口 |
| 登录限流 | 单 IP 5 分钟内最多 10 次登录尝试 | 防止暴力破解 |
| 上传限流 | 单用户 1 小时内最多 50 次上传 | 防止资源滥用 |
| 导出限流 | 单用户 1 小时内最多 10 次导出 | 防止服务器负载过高 |

超限返回：`{ code: 429, msg: "请求过于频繁，请稍后再试" }`

### 5.3 图片上传安全

| 项目 | 规范 |
|------|------|
| 大小限制 | 单张图片 < 5MB |
| 格式限制 | 仅允许 jpg、jpeg、png、webp |
| 格式校验 | 服务端读取文件头 Magic Number 校验，不依赖扩展名 |
| 存储路径 | 按日期分目录：`/data/uploads/recycle/YYYYMM/` |
| 文件名规则 | 时间戳 + 随机字符串 + 扩展名，禁止保留原始文件名 |
| 压缩处理 | 上传后自动生成缩略图（最大边 400px），原图单独存储 |
| 访问控制 | 图片 URL 需携带有效 Token 或设置合理的 Nginx 访问规则 |

### 5.4 数据脱敏

| 数据类型 | 脱敏规则 | 示例 |
|----------|----------|------|
| 手机号 | 中间 4 位替换为 * | `13812345678` → `138****5678` |
| 身份证号 | 中间 10 位替换为 * | `310101199001011234` → `310***********1234` |
| 真实姓名 | 仅保留姓氏 | `张三` → `张*` |
| 密码 | bcrypt 哈希存储，不可逆 | 永不返回明文或哈希值 |

**脱敏原则：**
- 列表接口默认返回脱敏数据
- 详情接口按当前用户权限决定是否脱敏
- 导出报表按用户角色决定是否包含完整信息
- 数据库存储原始数据，脱敏在序列化层处理

### 5.5 传输安全

| 项目 | 规范 |
|------|------|
| 协议 | 全站 HTTPS，禁止 HTTP 明文传输 |
| 证书 | Let's Encrypt 或云服务商证书，自动续期 |
| HSTS | 开启 HTTP Strict Transport Security |
| CORS | 仅允许管理后台域名跨域访问，禁止通配符 `*` |
| 请求签名 | 关键操作（如发放积分）需二次校验管理员密码 |

### 5.6 审计日志

| 项目 | 规范 |
|------|------|
| 记录范围 | 所有写操作（增删改）和敏感读操作 |
| 记录内容 | 操作人、操作时间、操作类型、目标对象、变更前/后数据、IP 地址 |
| 存储方式 | 异步写入数据库 operation_log 表 |
| 保留期限 | 日志保留 180 天，超期自动归档 |
| 防篡改 | 日志表禁止 UPDATE/DELETE，仅支持 INSERT |

            expires 30d;
            add_header Cache-Control "public, immutable";
        }
    }

    # API 反向代理
    location /api/ {
        proxy_pass http://backend:8000/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时配置

        proxy_send_timeout 30s;
        proxy_read_timeout 60s;
        
        # 文件上传大小限制
        client_max_body_size 50M;
    }

    # 图片资源访问

    }

    # 健康检查
    location /health {
        access_log off;
        return 200 "OK\n";
        add_header Content-Type text/plain;
    }
}

## 附录 A：接口状态流转

```
【创建回收单】        【管理员审核】        【仓库入库】         【积分发放】
   upload               audit               confirm              award
     │                    │                    │                    │
     ▼                    ▼                    ▼                    ▼
┌─────────┐         ┌─────────┐         ┌─────────┐         ┌─────────┐
│ pending │  →通过→  │approved │  →确认→  │ stored  │  →发放→  │completed│
│ 待审核  │  →驳回→  │ 已通过  │         │ 已入库  │         │ 已完成  │
└─────────┘         └─────────┘         └─────────┘         └─────────┘
     │                    │
     │              ┌─────┘
     │              │
     └────驳回────→ ▼
                ┌─────────┐
                │rejected │
                │ 已驳回  │
                └─────────┘
```

| 当前状态 | 可执行操作 | 操作后状态 |
|----------|------------|------------|
| 待审核 (pending) | 审核通过 | 已通过 (approved) |
| 待审核 (pending) | 审核驳回 | 已驳回 (rejected) |
| 已通过 (approved) | 入库确认 | 已入库 (stored) |
| 已入库 (stored) | 积分发放 | 已完成 (completed) |
| 已驳回 (rejected) | 重新上传（新建单） | 待审核 (pending) |
