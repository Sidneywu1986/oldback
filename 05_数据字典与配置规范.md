> 文档编号：WD-RECYCLE-DB-005
> 关联文档：《维修接单软件需求规格说明书》
> 适用范围：废旧配件回收子系统（Recycle Module）

## 1. 数据库设计规范

### 1.1 基础规范

| 规范项 | 要求值 | 说明 |
|--------|--------|------|
| 字符集 | `utf8mb4` | 支持完整 Unicode（含 Emoji） |
| 排序规则 | `utf8mb4_unicode_ci` | 大小写不敏感 |
| 存储引擎 | `InnoDB` | 支持事务、行级锁、外键 |
| 命名规范 | 小写下划线法（snake_case） | 如 `recycle_order`、`create_time` |
| 表前缀 | 无统一前缀，按模块命名 | 回收模块以 `recycle_` / `points_` / `parts_` 开头 |
| 时间存储 | `DATETIME` 类型 | 默认 `CURRENT_TIMESTAMP`，自动更新 `ON UPDATE` |
| 金额存储 | `DECIMAL(10,2)` | 精确小数，避免浮点误差 |
| 状态字段 | `TINYINT` | 枚举值需有 `COMMENT` 注释 |
| 逻辑删除 | 软删除，不使用物理删除 | 通过 `status` 字段控制 |

### 1.2 必须字段（Global Columns）

所有业务表必须包含以下字段：

| 字段名 | 类型 | 约束 | 说明 |
|--------|------|------|------|
| `id` | `BIGINT` | `PRIMARY KEY AUTO_INCREMENT` | 主键，全局唯一 |
| `create_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP` | 创建时间 |
| `update_time` | `DATETIME` | `DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP` | 更新时间 |

### 1.3 索引规范

| 索引类型 | 建索引条件 | 命名规范 |
|----------|-----------|----------|
| 主键索引 | 每个表必须 | 默认 `PRIMARY` |
| 唯一索引 | 业务唯一字段 | `uk_{字段名}` |
| 普通索引 | 高频查询条件、外键字段 | `idx_{字段名}` |
| 联合索引 | 组合查询条件 | `idx_{字段1}_{字段2}` |

### 1.4 字段注释规范

- 所有字段必须有 `COMMENT` 注释
- 枚举字段注释格式：`COMMENT '0-状态A 1-状态B 2-状态C'`
- 外键字段注释需标明关联表：`COMMENT '关联 master 表'`

## 2. 核心表结构

### 2.1 user —— 用户表（平台管理员）

存储平台后台管理用户，负责回收订单审核、积分管理、系统配置。

#### 字段说明

| 字段名 | 数据类型 | 可空 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | NO | AUTO_INCREMENT | 主键 |
| username | VARCHAR(32) | NO | - | 登录账号，唯一 |
| password_hash | VARCHAR(128) | NO | - | 密码哈希（bcrypt） |
| name | VARCHAR(32) | YES | NULL | 真实姓名 |
| role | TINYINT | NO | 0 | 0-普通管理员 1-超级管理员 2-审核专员 |
| status | TINYINT | NO | 1 | 0-禁用 1-启用 |
| create_time | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

#### DDL

```sql
CREATE TABLE `user` (
  `id`            BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `username`      VARCHAR(32) NOT NULL COMMENT '登录账号',
  `password_hash` VARCHAR(128) NOT NULL COMMENT '密码哈希（bcrypt）',
  `name`          VARCHAR(32) DEFAULT NULL COMMENT '真实姓名',
  `role`          TINYINT NOT NULL DEFAULT 0 COMMENT '0-普通管理员 1-超级管理员 2-审核专员',
  `status`        TINYINT NOT NULL DEFAULT 1 COMMENT '0-禁用 1-启用',
  `create_time`   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time`   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`),
  KEY `idx_role_status` (`role`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='平台管理员用户表';
```

### 2.3 recycle_order —— 回收订单表（核心表）

记录废旧配件回收的全生命周期，从师傅提交到最终处置。

#### 字段说明

| 字段名 | 数据类型 | 可空 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | NO | AUTO_INCREMENT | 主键 |
| order_no | VARCHAR(32) | NO | - | 回收单号，唯一（REC + 年月日 + 6位流水） |
| master_id | BIGINT | NO | - | 关联师傅表 |
| parts_name | VARCHAR(64) | YES | NULL | 配件名称（如压缩机、主板） |
| parts_type | VARCHAR(32) | YES | NULL | 配件类型（如金属/塑料/电路板） |
| device_type | VARCHAR(32) | YES | NULL | 设备类型（如空调/冰箱/洗衣机） |
| old_parts_img_url | VARCHAR(255) | YES | NULL | 旧配件照片URL |
| new_parts_img_url | VARCHAR(255) | YES | NULL | 新配件照片URL |
| work_img_url | VARCHAR(255) | YES | NULL | 维修完工照片URL |
| status | TINYINT | NO | 0 | 0-待审核 1-已通过 2-已驳回 3-已入库 4-已处置 |
| user_keep | TINYINT | NO | 0 | 0-平台回收 1-用户自留 |
| reject_reason | VARCHAR(255) | YES | NULL | 驳回原因 |
| points | INT | NO | 0 | 获得积分数量 |
| create_time | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |
| update_time | DATETIME | NO | CURRENT_TIMESTAMP ON UPDATE | 更新时间 |

#### DDL

```sql
CREATE TABLE `recycle_order` (
  `id`                BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `order_no`          VARCHAR(32) NOT NULL COMMENT '回收单号，格式 REC + yymmdd + 6位流水',
  `master_id`         BIGINT NOT NULL COMMENT '关联 master 表',
  `parts_name`        VARCHAR(64) DEFAULT NULL COMMENT '配件名称，如压缩机、主板',
  `parts_type`        VARCHAR(32) DEFAULT NULL COMMENT '配件类型，如金属/塑料/电路板',
  `device_type`       VARCHAR(32) DEFAULT NULL COMMENT '设备类型，如空调/冰箱/洗衣机',
  `old_parts_img_url` VARCHAR(255) DEFAULT NULL COMMENT '旧配件照片URL',
  `new_parts_img_url` VARCHAR(255) DEFAULT NULL COMMENT '新配件照片URL',
  `work_img_url`      VARCHAR(255) DEFAULT NULL COMMENT '维修完工照片URL',
  `status`            TINYINT NOT NULL DEFAULT 0 COMMENT '0-待审核 1-已通过 2-已驳回 3-已入库 4-已处置',
  `user_keep`         TINYINT NOT NULL DEFAULT 0 COMMENT '0-平台回收 1-用户自留',
  `reject_reason`     VARCHAR(255) DEFAULT NULL COMMENT '驳回原因',
  `points`            INT NOT NULL DEFAULT 0 COMMENT '获得积分数量',
  `create_time`       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time`       DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_order_no` (`order_no`),
  KEY `idx_master_id` (`master_id`),
  KEY `idx_status` (`status`),
  KEY `idx_parts_type` (`parts_type`),
  KEY `idx_create_time` (`create_time`),
  KEY `idx_status_create_time` (`status`, `create_time`)

### 2.4 recycle_audit —— 回收审核记录表

记录每一次回收订单的审核操作，形成完整的审计追踪链。

#### 字段说明

| 字段名 | 数据类型 | 可空 | 默认值 | 说明 |
|--------|----------|------|--------|------|
| id | BIGINT | NO | AUTO_INCREMENT | 主键 |
| recycle_id | BIGINT | NO | - | 关联 recycle_order 表 |
| auditor_id | BIGINT | NO | - | 关联 user 表（审核人） |
| action | VARCHAR(16) | NO | - | 操作类型：pass（通过）/ reject（驳回） |
| remark | VARCHAR(255) | YES | NULL | 审核备注 |
| create_time | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |

#### DDL

```sql
CREATE TABLE `recycle_audit` (
  `id`          BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `recycle_id`  BIGINT NOT NULL COMMENT '关联 recycle_order 表',
  `auditor_id`  BIGINT NOT NULL COMMENT '审核人ID，关联 user 表',
  `action`      VARCHAR(16) NOT NULL COMMENT '操作类型：pass-通过 reject-驳回',
  `remark`      VARCHAR(255) DEFAULT NULL COMMENT '审核备注',
  `create_time` DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  PRIMARY KEY (`id`),
  KEY `idx_recycle_id` (`recycle_id`),
  KEY `idx_auditor_id` (`auditor_id`),
  KEY `idx_action` (`action`),
  KEY `idx_create_time` (`create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='回收审核记录表';
```

### 2.5 points_record —— 积分记录表

| balance_after | INT | NO | - | 变动后余额 |
| remark | VARCHAR(255) | YES | NULL | 备注说明 |
| create_time | DATETIME | NO | CURRENT_TIMESTAMP | 创建时间 |

#### DDL

```sql
CREATE TABLE `points_record` (
  `id`            BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',

  `create_time`   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',

  KEY `idx_recycle_id` (`recycle_id`),
  KEY `idx_points_type` (`points_type`),
  KEY `idx_create_time` (`create_time`),
  KEY `idx_master_create_time` (`master_id`, `create_time`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='积分记录表';

## 3. 完整建库脚本（一键执行）

```sql
-- ========================================================
-- 废旧回收管理程序 —— 数据库初始化脚本
-- 执行前请确认：MySQL >= 8.0, InnoDB 引擎
-- ========================================================

SET NAMES utf8mb4;
SET FOREIGN_KEY_CHECKS = 0;

-- 1. 用户表（平台管理员）
CREATE TABLE IF NOT EXISTS `user` (
  `id`            BIGINT NOT NULL AUTO_INCREMENT COMMENT '主键',
  `username`      VARCHAR(32) NOT NULL COMMENT '登录账号',
  `password_hash` VARCHAR(128) NOT NULL COMMENT '密码哈希（bcrypt）',
  `name`          VARCHAR(32) DEFAULT NULL COMMENT '真实姓名',
  `role`          TINYINT NOT NULL DEFAULT 0 COMMENT '0-普通管理员 1-超级管理员 2-审核专员',
  `status`        TINYINT NOT NULL DEFAULT 1 COMMENT '0-禁用 1-启用',
  `create_time`   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
  `update_time`   DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
  PRIMARY KEY (`id`),
  UNIQUE KEY `uk_username` (`username`),
  KEY `idx_role_status` (`role`, `status`)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='平台管理员用户表';

-- 2. 师傅表
CREATE TABLE IF NOT EXISTS `master` (

    """系统配置读取工具类"""
    
    @staticmethod
    def get(key: str, default=None):
        """读取配置项，不存在返回默认值"""

    @staticmethod
    def get_int(key: str, default=0):
        """读取整数类型配置"""
        val = SystemConfig.get(key)
        return int(val) if val else default
    
    @staticmethod
    def get_float(key: str, default=0.0):
        """读取浮点类型配置"""
        val = SystemConfig.get(key)
        return float(val) if val else default

## 5. 状态机定义

### 5.1 回收订单状态流转图

```
┌─────────────────────────────────────────────────────────────────┐
│                    回收订单状态机（State Machine）                 │
└─────────────────────────────────────────────────────────────────┘

【状态节点】

  ┌──────────────┐     ┌──────────────┐     ┌──────────────┐
  │   0-待审核    │────▶│   1-已通过    │────▶│   3-已入库    │
  │  (PENDING)   │     │  (APPROVED)  │     │  (STORAGED)  │
  └──────┬───────┘     └──────────────┘     └──────┬───────┘
         │                                           │
         │         ┌───────────────────────────────┘
         │         │
         │    ┌────▼───────┐     ┌──────────────┐
         └───▶│   2-已驳回    │     │   4-已处置    │
               │  (REJECTED)  │     │ (DISPOSED)   │
               └──────────────┘     └──────────────┘

【状态说明】

| 状态值 | 状态名称 | 说明 | 允许的操作 |
|--------|----------|------|-----------|
| 0 | 待审核 | 师傅提交回收申请后初始状态 | 审核通过、审核驳回 |
| 1 | 已通过 | 管理员审核通过，待入库 | 确认入库 |
| 2 | 已驳回 | 审核不通过，需师傅重新提交 | 重新提交（创建新单） |
| 3 | 已入库 | 配件已存入仓库 | 确认处置 |
| 4 | 已处置 | 配件已完成最终处置（拆解/再利用/环保处置） | 无（终态） |

【状态流转规则】

| 流转路径 | 触发条件 | 执行者 | 伴随操作 |
|----------|---------|--------|---------|
| 0 → 1 | 审核通过 | 审核专员/自动审核 | 积分增加、写 audit 记录 |
| 0 → 2 | 审核驳回 | 审核专员 | 写 audit 记录、填写驳回原因 |
| 1 → 3 | 仓库确认入库 | 库管员/管理员 | 更新库存记录 |
| 3 → 4 | 完成环保处置 | 管理员 | 记录处置方式与时间 |

【不允许的流转（需拦截）】

| 非法流转 | 拦截原因 |
|----------|---------|
| 2 → 1 | 已驳回订单不可直接通过，需重新提交 |
| 4 → 任意 | 已处置为终态，不可回退 |
| 3 → 2 | 已入库不可驳回，需走退库流程 |
| 1 → 4 | 必须先入库再处置，不可跳过 |

【状态校验伪代码】

```python
VALID_TRANSITIONS = {
    0: [1, 2],    # 待审核 -> 已通过/已驳回
    1: [3],       # 已通过 -> 已入库
    2: [],        # 已驳回 -> 终态（不可流转）
    3: [4],       # 已入库 -> 已处置
    4: [],        # 已处置 -> 终态
}

def can_transition(current_status: int, target_status: int) -> bool:
    """校验状态流转是否合法"""
    return target_status in VALID_TRANSITIONS.get(current_status, [])

def transition(recycle_id: int, from_status: int, to_status: int, operator_id: int):
    """执行状态流转"""
    if not can_transition(from_status, to_status):
        raise ValueError(f"非法状态流转: {from_status} -> {to_status}")
    
    with db.transaction():
        # 1. 更新订单状态
        update_recycle_order_status(recycle_id, to_status)
        
        # 2. 记录审核日志
        insert_recycle_audit(recycle_id, operator_id, action, remark)
        
        # 3. 状态为"已通过"时发放积分
        if to_status == 1:
            grant_points(recycle_id)
```
